    	<h1 class="title">
    							iConsole
    	</h1><center><hr /></center>
    	<div class="Unindented">
    						iConsole users can manage <a class="Reference" href="#sec:Account">Account</a> settings and <a class="Reference" href="#sec:AddData">Add Data</a> , with left side menu it is simple to do. Admin users, those with IADMIN role, additionally administrate <a class="Reference" href="#sec:UserManagement">Users</a>, <a class="Reference" href="#sec:Roles">Roles</a> and <a class="Reference" href="#sec:Sources">Sources</a>. 
    					</div>
    	<h1 class="Section-">
    		<a class="toc" name="toc-Section--1"></a><span class="versalitas"><a class="Label" name="sec:Account"> </a></span>Account
    	</h1>
    	<div class="Unindented">
    		Account properties such as password and e-mail can be updated by users. 
    	</div>
    	<h2 class="Subsection-">
    		<a class="toc" name="toc-Subsection--1"></a>Change Password
    	</h2>
    	<ol>
    		<li>
    			Select <b>Change Password</b> from the iConsole Menu.
    		</li>
    		<li>
    			Register password information.<ul>
    		<li>
    			<b>Current password </b>
    		</li>
    		<li>
    			<b>New password </b>and<b> confirm.</b>
    		</li>

    		</ul>

    		</li>
    		<li>
    			Click &lt;&lt;Save&gt;&gt;.
    		</li>

    	</ol>
    	<div class="Unindented">
    		<div class="Boxed" style="width: 35%;">
    			<div class="left">
    				Note: Password requires at least 8 characters.
    			</div>

    		</div>

    	</div>
    	<h2 class="Subsection-">
    		<a class="toc" name="toc-Subsection--2"></a>Change E-mail
    	</h2>
    	<ol>
    		<li>
    			Select <b>Change Email</b> from the iConsole Menu.
    		</li>
    		<li>
    			Register e-mail information<ul>
    		<li>
    			<b>Password</b>
    		</li>
    		<li>
    			<b>New e-mail </b>and<b> confirm</b>
    		</li>

    		</ul>

    		</li>
    		<li>
    			Click &lt;&lt;Save&gt;&gt;.
    		</li>

    	</ol>
    	<h1 class="Section-">
    		<a class="toc" name="toc-Section--2"></a><a class="Label" name="sec:UserManagement"> </a>Users
    	</h1>
    	<div class="Unindented">
    		Set of concessions to consume the Intelinet and iConsole resources. Users may have a role or role but if none of these has sources user won’t be able to log into Intelinet.
    	</div>
    	<h2 class="Subsection-">
    		<a class="toc" name="toc-Subsection--3"></a><a class="Label" name="CreateUseriConsole"> </a><span class="black">Create User </span>
    	</h2>
    	<ol>
    		<li>
    			Select <b>Create User</b> from the iConsole.
    		</li>
    		<li>
    			Register user’s information:<ul>
    			<li>
    				<b>User</b> must be unique, only alphanumeric and underscore characters allowed. This is the end user’s login name. Automatically converted to lowercase.
    			</li>
    			<li>
    				<b>First Name</b> mandatory. Automatically converts the first letter to uppercase.
    			</li>
    			<li>
    				<b>Last Name </b>mandatory.
    			</li>
    			<li>
    				<b>Mail</b> 
    			</li>
    			<li>	
    				<b>Status</b> enabled or disabled, when enabled a mail will be sent with user credentials, if disabled an activation mail.
    			</li>
    			<li>
    				<b>Roles </b>are necessary to feed the Intelinet search, minimum one role required.<b><br/>
    				</b><table>
    					<tr>
    						<td align="center" valign="top">
    							Authorized sources
    						</td>
    						<td align="center" valign="top">
    							Available sources
    						</td>

    					</tr>
    					<tr>
    						<td align="center" valign="top">
    							Roles to attach to the new user
    						</td>
    						<td align="center" valign="top">
    							Roles available
    						</td>

    					</tr>

    				</table>

    			</li>

    			</ul>

    		</li>
    		<li>
    			Click &lt;&lt;Save&gt;&gt; to finish registration.
    		</li>

    	</ol>
    	<div class="Subparagraph">
    		<div class="Boxed" style="width: 35%;">
    			Disabled users can’t log into Intelinet until the activation process is complete.
    		</div>

    	</div>
    	<h2 class="Subsection-">
    		<a class="toc" name="toc-Subsection--4"></a><a class="Label" name="ListUsers/EditRoles"> </a><span class="black">List Users/Edit Roles</span>
    	</h2>
    	<ol>
    		<li>
    			Select <b>List Users</b> from the iConsole Menu.
    		</li>
    		<li>
    			All users are listed alphabetically, and can be filtered by status (automatically) and/or keyword registered in the search box and click &lt;&lt;Search&gt;&gt;.
    		</li>
    		<li>
    			Click <img class="embedded" src="Images/edit.png" alt="figure Images/edit.png" style="width: 32px; max-width: 27px; height: 34px; max-height: 29px;"/>
    			to edit roles related to the selected user.
    		</li>
    		<li>
    			Edit Assigned roles and click &lt;&lt;Save&gt;&gt;.
    		</li>

    	</ol>
    	<div class="Unindented">
    		Click <img class="embedded" src="Images/broom.png" alt="figure Images/broom.png" style="width: 30px; max-width: 38px; height: 20px; max-height: 25px;"/>
    		 to reset search parameters.
    	</div>
    	<h2 class="Subsection-">
    		<a class="toc" name="toc-Subsection--5"></a><a class="Label" name="EnableUsers"> </a><span class="black">Enable Users </span>
    	</h2>
    	<ol>
    		<li>
    			Select <b>Enable Users</b> from the iConsole Menu.
    		</li>
    		<li>
    			Disabled users are listed, check those you want to enable. 
    		</li>
    		<li>
    			Click &lt;&lt;Enable&gt;&gt; button, confirm.
    		</li>

    	</ol>
    	<h2 class="Subsection-">
    		<a class="toc" name="toc-Subsection--6"></a><a class="Label" name="DisableUsers"> </a><span class="black">Disable Users</span>
    	</h2>
    	<ol>
    		<li>
    			Select <b>Disable User</b> from the iConsole Menu.
    		</li>
    		<li>
    			Enabled users are listed, check those you want to disable.
    		</li>
    		<li>
    			Click &lt;&lt;Disable&gt;&gt; button, confirm.
    		</li>

    	</ol>
    	<div class="Unindented">
    		Note: Logged user cannot disable himself and will not appear on the list.
    	</div>
    	<h2 class="Subsection-">
    		<a class="toc" name="toc-Subsection--7"></a><a class="Label" name="SendActivationMail"> </a><span class="black">Send activation mail</span>
    	</h2>
    	<ol>
    		<li>
    			Select <b>Send activation mail</b> from the iConsole Menu.
    		</li>
    		<li>
    			Disabled new users are listed, check those you want to activate.
    		</li>
    		<li>
    			Click &lt;&lt;Send e-mail&gt;&gt; button, confirm.
    		</li>

    	</ol>
    	<h2 class="Subsection-">
    		<a class="toc" name="toc-Subsection--8"></a><a class="Label" name="DeleteUser"> </a><span class="black">Delete Users</span>
    	</h2>
    	<ol>
    		<li>
    			Select <b>Delete Users</b> from the iConsole Menu.
    		</li>
    		<li>
    			All users are listed, check those to be deleted.
    		</li>
    		<li>
    			Click the &lt;&lt;Delete&gt;&gt; button, confirm.
    		</li>

    	</ol>
    	<div class="Unindented">
    		<div class="Boxed" style="width: 35%;">
    			Note: A logged user cannot delete himself and will not appear on the list.
    		</div>

    	</div>
    	<h1 class="Section-">
    		<a class="toc" name="toc-Section--3"></a><a class="Label" name="sec:Roles"> </a>Roles
    	</h1>
    	<div class="Unindented">
    		Set of information sources available for a search.
    	</div>
    	<h2 class="Subsection-">
    		<a class="toc" name="toc-Subsection--9"></a>Create Role
    	</h2>
    	<ol>
    		<li>
    			Select <b>Create Role</b> from the iConsole menu.
    		</li>
    		<li>
    			Register role’s information<ul>
    			<li>
    				<b>Role name </b>automatically converted to lowercase.
    			</li>
    			<li>
    				<b>Sources</b> relate one or more sources to the new role,optional.<br/>
    				<table>
    					<tr>
    						<td align="center" valign="top">
    							<b><i>Authorized Sources</i></b>
    						</td>
    						<td align="center" valign="top">
    							<b><i>Available Sources</i></b>
    						</td>

    					</tr>
    					<tr>
    						<td align="center" valign="top">
    							Sources to relate to the new role.
    						</td>
    						<td align="center" valign="top">
    							List of all roles available for the new role.
    						</td>

    					</tr>

    				</table>

    			</li>

    			</ul>

    		</li>
    		<li>
    			Click &lt;&lt;Save&gt;&gt;.
    		</li>

    	</ol>
    	<h2 class="Subsection-">
    		<a class="toc" name="toc-Subsection--10"></a>List Roles
    	</h2>
    	<div class="Unindented">
    		Select <b>List Roles</b> from the iConsole Menu to view all users alphabetically. This grid contains the following features:
    	</div>
    	<ul>
    		<li>
    			Click <img class="embedded" src="Images/edit.png" alt="figure Images/edit.png" style="max-width: 27px; max-height: 29px;"/>
    			 to grant or revoke information sources. Sources in the <i>Authorized Sources</i> list will be available when selected role be assigned.y 
    		</li>
    		<li>
    			Click <img class="embedded" src="Images/delete.png" alt="figure Images/delete.png" style="max-width: 33px; max-height: 32px;"/>
    			to eliminate role, this operation also removes any former user-role relation, confirm.<br/>
    			Roles that may leave at least one orphan user, won’t be deleted.
    		</li>
    		<li>
    			Click <img class="embedded" src="Images/users.png" alt="figure Images/users.png" style="max-width: 33px; max-height: 28px;"/>
    			 to list all users that have assigned the selected role. (Read only)
    		</li>
    		<li>
    			Click <img class="embedded" src="Images/sources.png" alt="figure Images/sources.png" style="max-width: 31px; max-height: 29px;"/>
    			 to list all sources attached to the selected role. (Read only)
    		</li>

    	</ul>
    	<h1 class="Section-">
    		<a class="toc" name="toc-Section--4"></a><span class="versalitas"><a class="Label" name="sec:Sources"> </a></span>Sources
    	</h1>
    	<div class="Unindented">

    	</div>
    	<h2 class="Subsection-">
    		<a class="toc" name="toc-Subsection--11"></a>List Sources
    	</h2>
    	<div class="Unindented">
    		Select <b>List Sources</b> from the iConsole Menu to view all roles alphabetically ordered. 
    	</div>
    	<ul>
    		<li>
    			Click <img class="embedded" src="Images/enable.png" alt="figure Images/enable.png" style="max-width: 27px; max-height: 22px;"/>
    			 to enable selected source, confirm.
    		</li>
    		<li>
    			Click <img class="embedded" src="Images/disable.png" alt="figure Images/disable.png" style="max-width: 23px; max-height: 22px;"/>
    			 to disable selected source, confirm.
    		</li>
    		<li>
    			Click<img class="embedded" src="Images/roles.png" alt="figure Images/roles.png" style="max-width: 32px; max-height: 30px;"/>
    			to list all related roles of the selected source. (Read only)
    		</li>

    	</ul>
    	<h1 class="Section-">
    		<a class="toc" name="toc-Section--5"></a><span class="versalitas"><a class="Label" name="sec:AddData"> </a></span>Add Data
    	</h1>
    	<div class="Unindented">
    		Users are authorized to upload archives for their enabled sources.
    	</div>
    	<ol>
    		<li>
    			Select <b>Add Data</b> from the iConsole Menu.
    		</li>
    		<li>
    			Register archives information<ul>
    			<li>
    				Select <b>information type</b>
    			.
    			</li>
    			<li>
    				Click <img class="embedded" src="Images/addArchive.png" alt="figure Images/addArchive.png" style="max-width: 26px; max-height: 31px;"/>
    				 and choose archive.
    			</li>
    			<li>
    				<b>Archive description</b>.
    			</li>
    		</ul>
    		</li>
    		<li>
    			Click &lt;&lt;Save&gt;&gt;.
    		</li>
    	</ol>
    	<div class="Unindented">
    		<div class="Boxed" style="width: 35%;">
    			Archives have no size or type restriction.
